Python SDK

Native Python client library for Pilot Protocol with CLI tools and type hints.

Package: pilotprotocol  •  PyPI: pypi.org/project/pilotprotocol  •  Requires: Python 3.10+

Installation

pip install pilotprotocol

This installs:

  • Python SDK — pilotprotocol module with Driver, DataExchange, EventStream classes
  • CLI Tools — pilotctl, pilot-daemon, pilot-gateway executables
  • Native Binaries — Platform-specific Go binaries bundled in the wheel

Quick Start

1. Start the Daemon

The daemon must be running before using the SDK:

pilot-daemon start --hostname my-agent

Or use the CLI wrapper:

from pilotprotocol.cli import run_daemon
# Starts daemon in subprocess (not typical usage)

2. Connect to the Daemon

from pilotprotocol import Driver

driver = Driver()
driver.connect()

# Get agent information
info = driver.info()
print(f"Address: {info['address']}")
print(f"Hostname: {info['hostname']}")
print(f"Public: {info['public']}")

driver.close()

3. Establish Connections

from pilotprotocol import Driver

driver = Driver()
driver.connect()

# Connect to a peer on port 1000 (stdio)
conn = driver.pilot_connect("other-agent", port=1000)

# Send data
conn.write(b"Hello from Python!")

# Receive response
data = conn.read(timeout=5.0)
print(f"Received: {data.decode()}")

conn.close()
driver.close()

API Reference

Driver

Main entry point for interacting with the Pilot Protocol daemon.

Constructor

driver = Driver(socket_path: str = "/tmp/pilot.sock")

Parameters:

  • socket_path — Path to the daemon's Unix socket (default: /tmp/pilot.sock)

Methods

connect()
driver.connect() -> None

Connects to the daemon. Must be called before any other operations.

Raises: RuntimeError if connection fails.

close()
driver.close() -> None

Closes the connection to the daemon. Always call this when done to release resources.

info()
driver.info() -> dict

Returns agent information.

Returns:

{
  "address": "0:0000.0000.0042",  # Virtual address
  "hostname": "my-agent",          # Hostname
  "public": false,                 # Public visibility
  "uptime": 3600                   # Uptime in seconds
}
pilot_connect()
driver.pilot_connect(addr: str, port: int = 1000) -> Connection

Establishes a connection to a peer.

Parameters:

  • addr — Peer address (hostname or virtual address)
  • port — Target port (default: 1000)

Returns: Connection object

Raises: RuntimeError on connection failure

pilot_listen()
driver.pilot_listen(port: int) -> Listener

Listens for incoming connections on a port.

Parameters:

  • port — Port to listen on

Returns: Listener object

resolve()
driver.resolve(hostname: str) -> str

Resolves a hostname to a virtual address.

Returns: Virtual address (e.g., "0:0000.0000.0042")

ping()
driver.ping(addr: str) -> dict

Pings a peer and returns RTT statistics.

Returns:

{
  "min": 12.3,     # Min RTT (ms)
  "avg": 15.7,     # Avg RTT (ms)
  "max": 18.2,     # Max RTT (ms)
  "count": 10      # Packets sent
}

Connection

Represents an active connection to a peer.

Methods

write()
conn.write(data: bytes) -> int

Writes data to the connection.

Returns: Number of bytes written

read()
conn.read(size: int = 65536, timeout: float = 10.0) -> bytes

Reads data from the connection.

Parameters:

  • size — Maximum bytes to read (default: 64KB)
  • timeout — Read timeout in seconds (default: 10s)

Returns: Received data

Raises: TimeoutError if no data received within timeout

close()
conn.close() -> None

Closes the connection.

Listener

Listens for incoming connections.

Methods

accept()
listener.accept(timeout: float = 60.0) -> Connection

Accepts an incoming connection.

Returns: Connection object

Raises: TimeoutError if no connection within timeout

close()
listener.close() -> None

Closes the listener.

DataExchange

High-level protocol for typed data exchange (port 1001).

Constructor

from pilotprotocol import DataExchange

de = DataExchange(driver: Driver)

Methods

connect()
de.connect(addr: str) -> None

Connects to a peer's data exchange service.

send_text()
de.send_text(text: str) -> None

Sends a text message.

send_json()
de.send_json(data: dict) -> None

Sends JSON data.

send_binary()
de.send_binary(data: bytes) -> None

Sends binary data.

send_file()
de.send_file(path: str) -> None

Sends a file.

receive()
de.receive(timeout: float = 10.0) -> dict

Receives a typed frame.

Returns:

{
  "type": "text",           # Frame type: text, json, binary, file
  "data": "...",            # Data (decoded based on type)
  "filename": "file.txt"    # Filename (only for file type)
}
close()
de.close() -> None

Closes the connection.

EventStream

Pub/sub event broker (port 1002).

Constructor

from pilotprotocol import EventStream

es = EventStream(driver: Driver)

Methods

connect()
es.connect(addr: str) -> None

Connects to a peer's event stream service.

subscribe()
es.subscribe(topic: str) -> None

Subscribes to a topic. Supports wildcards: *, **

Examples:

  • status — Exact match
  • sensor.* — Single-level wildcard
  • sensor/** — Multi-level wildcard
publish()
es.publish(topic: str, data: str) -> None

Publishes an event to a topic.

receive()
es.receive(timeout: float = 10.0) -> dict

Receives an event.

Returns:

{
  "topic": "sensor/temperature",
  "data": "23.5"
}
close()
es.close() -> None

Closes the connection.

Usage Examples

Echo Server

from pilotprotocol import Driver

driver = Driver()
driver.connect()

# Listen on port 5000
listener = driver.pilot_listen(5000)
print("Listening on port 5000...")

while True:
    conn = listener.accept()
    print("Connection accepted")
    
    data = conn.read()
    print(f"Received: {data.decode()}")
    
    conn.write(data)  # Echo back
    conn.close()
    print("Connection closed")

Send JSON Data

from pilotprotocol import Driver, DataExchange

driver = Driver()
driver.connect()

de = DataExchange(driver)
de.connect("other-agent")

# Send structured data
de.send_json({
    "command": "status",
    "timestamp": 1234567890,
    "metadata": {
        "version": "1.0"
    }
})

# Receive response
response = de.receive()
print(f"Response: {response}")

de.close()
driver.close()

Pub/Sub Events

from pilotprotocol import Driver, EventStream

driver = Driver()
driver.connect()

es = EventStream(driver)
es.connect("other-agent")

# Subscribe to all sensor topics
es.subscribe("sensor/**")

# Receive events
for i in range(10):
    event = es.receive(timeout=30.0)
    print(f"Topic: {event['topic']}")
    print(f"Data: {event['data']}")

es.close()
driver.close()

File Transfer

from pilotprotocol import Driver, DataExchange

driver = Driver()
driver.connect()

de = DataExchange(driver)
de.connect("other-agent")

# Send a file
de.send_file("/path/to/document.pdf")
print("File sent")

de.close()
driver.close()

Context Managers

All classes support context managers for automatic cleanup:

from pilotprotocol import Driver, DataExchange

with Driver() as driver:
    driver.connect()
    
    with DataExchange(driver) as de:
        de.connect("other-agent")
        de.send_text("Hello!")
        response = de.receive()
        print(response['data'])

Error Handling

The SDK raises Python exceptions for errors:

from pilotprotocol import Driver

driver = Driver()

try:
    driver.connect()
    conn = driver.pilot_connect("nonexistent-agent")
except RuntimeError as e:
    print(f"Error: {e}")
    # e.g., "Failed to connect: no route to host"
finally:
    driver.close()

Common exceptions:

  • RuntimeError — Connection failures, protocol errors
  • TimeoutError — Read/accept timeouts
  • ValueError — Invalid parameters

Type Hints

The SDK includes comprehensive type annotations:

from pilotprotocol import Driver, Connection
from typing import Dict

driver: Driver = Driver()
driver.connect()

conn: Connection = driver.pilot_connect("other-agent")
info: Dict[str, any] = driver.info()

# Type checkers (mypy, pyright) will validate usage

CLI Tools

The package includes CLI wrappers for the Go binaries:

  • pilotctl — Main CLI tool (equivalent to the Go binary)
  • pilot-daemon — Start the daemon
  • pilot-gateway — Start the IP gateway

These are standard Python console script entry points that execute the bundled binaries.

Platform Support

The SDK provides platform-specific wheels for:

  • Linux — x86_64 (manylinux)
  • macOS — x86_64, ARM64 (Apple Silicon)

Windows support is planned for future releases.

Architecture

The Python SDK uses ctypes to call Go functions exported via CGO:

  • FFI Layer — ctypes bindings to libpilot.so/.dylib
  • Memory Management — Automatic cleanup of Go-allocated strings
  • Handle Pattern — Opaque handles for connections, listeners
  • Error Handling — Go errors converted to Python exceptions
